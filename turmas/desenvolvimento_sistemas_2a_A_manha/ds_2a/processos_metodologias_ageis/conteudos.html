Plano de Curso – Processos de Desenvolvimento de Software

Este material apresenta uma sequência de aulas sobre requisitos, testes de software, métodos ágeis, DevOps e infraestrutura moderna. Cada semana possui um pequeno exercício para estimular o aprendizado prático.

Semana 1 – Introdução aos Requisitos

  • Aula 1: O que são requisitos de software?
  • Aula 2: Ciclo de vida do requisito
  • Aula 3: Tipos de requisitos
Exercício:

Escolha um sistema conhecido (ex: aplicativo de banco ou rede social) e escreva 3 requisitos funcionais e 3 requisitos não funcionais.

Semana 2 – Elicitação de Requisitos

  • Aula 1: O que é elicitação de requisitos?
  • Aula 2: Entrevistas e questionários
  • Aula 3: Observação e brainstorming
Exercício:

Crie 5 perguntas que você faria a um cliente para entender as necessidades de um sistema de biblioteca escolar.

Semana 3 – Manifesto Ágil

  • Aula 1: O que é o Manifesto Ágil?
  • Aula 2: Os 12 princípios ágeis
  • Aula 3: Valores ágeis na prática
Exercício:

Explique com suas palavras um dos quatro valores do Manifesto Ágil.

Semana 4 – Introdução aos Testes de Software

  • Aula 1: O que são testes de software?
  • Aula 2: Tipos de testes
  • Aula 3: Testes manuais x automatizados
Exercício:

Liste 3 possíveis erros que poderiam acontecer em um sistema de login.

Semana 5 – Testes Funcionais

  • Aula 1: O que são testes funcionais?
  • Aula 2: Técnicas de caso de teste
  • Aula 3: Execução e registro de resultados
Exercício:

Crie um caso de teste simples para um sistema de cadastro de usuários.

Semana 6 – Qualidade de Requisitos

  • Aula 1: Diferença entre verificação e validação
  • Aula 2: Critérios de qualidade de um requisito
  • Aula 3: Técnicas de revisão de requisitos
Exercício:

Analise o seguinte requisito: "O sistema deve ser rápido". Explique por que ele não está bem definido.

Semana 7 – Mudanças de Requisitos

  • Aula 1: Por que os requisitos mudam?
  • Aula 2: Processo de controle de mudanças
  • Aula 3: Avaliação de impacto da mudança
Exercício:

Descreva um exemplo de mudança em um sistema e qual impacto ela poderia causar no projeto.

Semana 8 – Casos de Teste e Automação

  • Aula 1: Estrutura de um caso de teste
  • Aula 2: Introdução à automação com Selenium IDE
  • Aula 3: Análise de resultados e registro de defeitos
Exercício:

Descreva os passos para testar o login de um sistema.

Semana 9 – Métricas de Qualidade

  • Aula 1: Indicadores de qualidade em testes
  • Aula 2: Ferramentas de métricas
  • Aula 3: Revisão de plano de teste
Exercício:

Explique por que métricas são importantes para avaliar a qualidade de software.

Semana 10 – Modelos de Desenvolvimento

  • Aula 1: Ciclo Waterfall clássico
  • Aula 2: Iterações curtas no ágil
  • Aula 3: Debate ágil x cascata
Exercício:

Liste uma vantagem e uma desvantagem do modelo cascata.

Semana 11 – Planejamento de Produto

  • Aula 1: Roadmap de Produto
  • Aula 2: Backlog e Sprints
  • Aula 3: Revisão de Sprint
Exercício:

Crie um pequeno backlog para um aplicativo de lista de tarefas.

Semana 12 – Papéis no Scrum

  • Aula 1: Product Owner
  • Aula 2: Scrum Master
  • Aula 3: Equipe de desenvolvimento
Exercício:

Explique a função de cada papel no Scrum.

Semana 13 – Métricas Ágeis

  • Aula 1: Velocity
  • Aula 2: Burndown Chart
  • Aula 3: Cumulative Flow Diagram
Exercício:

Pesquise um exemplo de gráfico Burndown e explique o que ele representa.

Semana 14 – Retrospectivas

  • Aula 1: Modelos de retrospectiva
  • Aula 2: Priorização de melhorias
  • Aula 3: Kaizen no desenvolvimento
Exercício:

Escreva duas melhorias que poderiam ser aplicadas após uma retrospectiva.

Semana 15 – SDLC

  • Aula 1: Fases clássicas do SDLC
  • Aula 2: Modelo Cascata
  • Aula 3: Modelos Incrementais
Exercício:

Desenhe um fluxo simples das fases do desenvolvimento de software.

Semana 16 – Scrum na prática

  • Aula 1: Valores do Manifesto Ágil
  • Aula 2: Papéis Scrum
  • Aula 3: Dinâmica de produto em papel
Exercício:

Projete uma ideia simples de aplicativo e descreva seu objetivo.

Semana 17 – Eventos Scrum

  • Aula 1: Sprint Planning
  • Aula 2: Daily Scrum
  • Aula 3: Review e Retrospective
Exercício:

Simule uma reunião Daily Scrum com três perguntas básicas.

Semana 18 – Kanban

  • Aula 1: Fundamentos do Kanban
  • Aula 2: Políticas explícitas
  • Aula 3: Simulação de fluxo de trabalho
Exercício:

Crie um quadro Kanban simples com três colunas: Fazer, Fazendo e Concluído.

Semana 19 – Estimativas

  • Aula 1: Story Points
  • Aula 2: Velocity
  • Aula 3: Planejamento de Release
Exercício:

Estime três tarefas usando Story Points (1, 3 ou 5).

Semana 20 – Integração Contínua

  • Aula 1: Conceito de CI
  • Aula 2: GitHub Actions
  • Aula 3: Deploy automatizado
Exercício:

Explique por que integrar código frequentemente reduz erros.

Semana 21 – DevOps

  • Aula 1: Origem do DevOps
  • Aula 2: Pipeline de feedback
  • Aula 3: Métricas DORA
Exercício:

Pesquise o significado de CI/CD e escreva um resumo.

Semana 22 – Infraestrutura como Código

  • Aula 1: Conceitos de Terraform
  • Aula 2: Módulos e providers
  • Aula 3: Laboratório de infraestrutura
Exercício:

Explique o que significa infraestrutura como código.

Semana 23 – Containers

  • Aula 1: Conceitos de containers
  • Aula 2: Dockerfile
  • Aula 3: Redes e logs
Exercício:

Explique a diferença entre container e máquina virtual.

Semana 24 – Kubernetes

  • Aula 1: Arquitetura Kubernetes
  • Aula 2: Deployment e Services
  • Aula 3: Escalabilidade
Exercício:

Explique por que escalabilidade é importante em sistemas modernos.

Semana 25 – SRE

  • Aula 1: Papel do SRE
  • Aula 2: SLI e SLO
  • Aula 3: Error Budget
Exercício:

Explique como confiabilidade impacta a experiência do usuário.

Semana 26 – Segurança em Software

  • Aula 1: Shift-Left Security
  • Aula 2: Segurança em containers
  • Aula 3: Análise de vulnerabilidades
Exercício:

Liste três boas práticas para manter sistemas seguros.

Material educacional aberto para estudo de desenvolvimento de software. Os estudantes são incentivados a modificar, testar e criar novos exemplos.
